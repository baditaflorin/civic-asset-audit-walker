# 0070 Documentation Reality Alignment

- Status: accepted

## Context

Several README claims are technically true but too broad for the current user experience.

## Decision

README claims must map to tested behavior. Limitations are documented explicitly when browser APIs or Mode A constraints shape the workflow.

## Consequences

Documentation becomes a contract rather than aspirational copy.

## Alternatives considered

- Keep README optimistic: rejected because Phase 3 is specifically about eliminating that drift.
